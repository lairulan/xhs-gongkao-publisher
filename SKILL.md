# 小红书公考种草发布助手 v1.2.0

自动生成公考备考种草内容并发布到小红书。支持信息图（干货/攻略）和文字笔记（经验/心态）两种形式，每天 1-2 篇，单账号运营。

### Step 0: 环境预检

<!-- 🔧 修改指引：
     - 如果 MCP 服务路径变了，改下面的路径
     - 如果换了配图工具（比如不用 gemini-image），改第3项

**必须在开始前完成，任何一项失败则停止并提示用户。**

1. 检查 xiaohongshu-mcp 服务是否可用
   - 尝试调用 MCP 工具 `check_login_status`
   - 如果服务未启动，提示用户运行：`~/xiaohongshu-mcp/xiaohongshu-mcp-darwin-arm64`
2. 检查登录状态
   - 已登录：显示用户名，继续
   - 未登录：调用 `get_login_qrcode` 获取二维码，提示用户扫码
3. 检查配图能力
   - 确认 `baoyu-xhs-images` Skill 可用（信息图模式）
   - 确认 `gemini-image` 或 `baoyu-image-gen` Skill 可用（文字笔记配图）

### Step 1: 智能选题

<!-- 🔧 修改指引：
     - 添加/调整搜索关键词：改下面的关键词列表
     - 调整优先级逻辑：交换优先级1/2/3的顺序
     - 新增选题来源：在优先级列表中添加新的来源（如RSS、API等）
     - 关键词池扩展：同步更新 content-pillars.md 中的搜索关键词 -->

按优先级依次尝试，选出 1 个最佳选题：

**优先级 1：时事热点关联公考**
- 使用 WebSearch 搜索最近 3 天公考相关新闻
- 搜索关键词：`公务员考试 最新消息 {当前年份}`、`国考省考 政策变化`、`申论热点话题`
- 如果有重大新闻（报名时间、政策变化、考纲调整），立即锁定此选题

**优先级 2：小红书热门笔记分析**
- 调用 `search_feeds` 搜索以下关键词（轮换）：
  - `公考上岸经验`、`行测秒杀技巧`、`申论高分模板`、`公考备考时间表`
  - `公务员岗位推荐`、`公考资料推荐`、`在职备考公务员`
- 取前 5 条高互动笔记，分析其选题方向和内容结构
- 选择一个可以"差异化"或"补充"的角度

**优先级 3：内容支柱自动轮换**
- 根据当天日期和 `references/content-pillars.md` 中的轮换表自动选择
- 确保相邻两天的内容方向不重复

**选题产出**：
```
选题：[具体主题]
方向：[6大方向之一]
笔记形式：[信息图 / 文字笔记]
关键词：[3-5个搜索关键词]
参考笔记：[如有，列出参考的热门笔记标题]
```

### Step 2: 竞品笔记分析（可选）

<!-- 🔧 修改指引：
     - 此步骤仅在 --viral 模式或用户提供笔记链接时触发
     - 5维度拆解模板可根据运营经验调整权重
     - 后续可考虑自动化：定期爬取竞品数据存入本地库 -->

当用户指定"爆款复刻"模式，或提供了某个笔记链接时执行：

1. 用 `search_feeds` 搜索同类笔记，或用 `get_feed_detail` 获取指定笔记详情
2. 拆解爆款 5 维度模板：
   - **标题模板**：句式（数字+痛点？疑问？感叹？）、字数、关键词位置
   - **封面模板**：文字信息层级、配色风格
   - **正文模板**：开场钩子、段落数、结尾 CTA 类型
   - **互动模板**：评论区引导方式（提问？投票？晒经验？）
   - **话题模板**：话题标签数量和类型
3. 产出改写方向（保留结构，替换具体内容）

### Step 3: 内容生成

<!-- 🔧 修改指引：
     - 这是核心分流点！根据 Step 1 产出的"笔记形式"走不同模式
     - 模式 A（信息图）：产出草稿→卡片→配文，草稿最终传给 baoyu-xhs-images
     - 模式 B（文字笔记）：产出文案→配图prompt→标签，文案直接用于发布
     - 语气规则来自 persona.md，文案格式来自 xhs-copywriting-rules.md
     - 字数范围可根据实际效果微调 -->

**遵守 persona.md 中的语气和说话方式。** 遵守 `references/xhs-copywriting-rules.md` 中的文案规则。

#### 【模式 A：信息图笔记】适用于干货/种草/岗位解析

1. **生成完整内容草稿**（1200-1800 字）
   - 围绕选题展开，结构清晰，干货密集
   - 参考 persona.md 的语气

2. **拆分为 6-9 张信息图卡片**
   每张卡片包含：
   - 卡片序号和小标题
   - 核心内容（80-150 字，要点精炼）
   - 视觉元素建议（图标/插画/数据）

   卡片结构模板：
   ```
   卡 1（封面）：大标题 + 钩子副标题 + 账号标识
   卡 2-7（内容卡）：小标题 + 要点（3-5条）+ 图标/插画
   卡 8（总结/CTA）：一句话总结 + 互动引导 + "关注我获取更多公考干货"
   ```

3. **生成小红书配文**（300-500 字）
   - 开头钩子（1-2句，制造好奇/共鸣）
   - 简要概括图片内容（不重复，而是补充）
   - 互动引导（提问/征集经验）
   - 话题标签 5-8 个

#### 【模式 B：文字笔记】适用于经验/心态/热点

1. **生成小红书文案**（500-800 字）
   - 短段落（2-3 句一段）
   - 多换行，段落间空一行
   - 适度使用 emoji（每段 0-2 个）
   - 口语化、有共鸣感
   - 第一句必须是钩子（反转/数据/提问）

2. **配图 prompt**（1-3 张）
   - 场景类：书桌、笔记本、咖啡、图书馆
   - 文字类：手写笔记风格、备考计划表
   - 风格：清新、文艺、学生感

3. **话题标签** 5-8 个

### Step 3.5: 人性化改写（反 AI 检测）

<!-- ⚠️ 关键步骤！v1.1.0 新增，强制执行不可跳过
     背景：小红书 AIGC 检测准确率 ~95%，月均拦截 10 万篇
     三层防御：内容层（8条规则）+ 图片层（写实风格）+ 行为层（见 publish-sop.md）
     🔧 修改指引：
     - 8 条规则的具体操作可以根据实际被检测的反馈微调
     - 6 项自检清单是最低标准，通过运营可以逐步收紧/放松
     - 图片策略：信息图（排版图）安全；AI 绘画需要写实化处理
     - 详细的"人味"注入规则见 xhs-copywriting-rules.md 的"去AI味写作规则"章节
     - 个人细节素材库见 persona.md 的"个人细节素材库"表格 -->

**此步骤强制执行，不可跳过。** 小红书有严格的 AIGC 检测机制（文本分类器准确率 ~95%、图像像素分析、行为模式识别），月均拦截 10 万篇 AI 低质笔记。

#### 文本去 AI 化（8 条硬规则）

对 Step 3 生成的初稿执行以下改写：

1. **消灭承接词**：删除所有段落开头的"首先/其次/然后/此外/总之/综上/最后/接下来/与此同时"，段落间靠内容自然衔接
2. **打破完美逻辑**：允许适当的话题跳跃，不要每段都严丝合缝地递进。可以突然插一句"对了说到这个..."、"哦差点忘了..."
3. **注入不完美表达**：加入省略号中断("真的好用...怎么说呢就是那种")、自我纠正("不对，应该说是...")、犹豫语气("大概是...我记得好像是")
4. **句式长短交替**：严禁连续 3 句以上相同句式长度。交替使用短句(5-10字)和长句(15-25字)
5. **加入具体私人细节**：每篇至少 2 处不可复制的个人化细节（"我当时在星巴克复习突然想到的"、"我室友看了都说绝了"）
6. **随机口语化替换**：将部分书面语替换为口语（"建议" → "我觉得吧"、"提高效率" → "学得快"、"值得注意的是" → "有个坑要注意"）
7. **情绪节奏波动**：不要全程正能量或全程干货，穿插真实情绪（吐槽、自嘲、小得意、小崩溃）
8. **去除模板化结尾**：不要用"希望对大家有帮助"、"以上就是我的分享"等 AI 典型结尾，换成更随意的收尾（"就说这么多吧累了"、"先写到这 想到再补充"、"有啥问题评论区见"）

#### 改写前后对比自检

改写完成后，对照检查：
- [ ] 全文无"首先/其次/然后/此外/总之"等承接词开头
- [ ] 至少 2 处个人化细节（时间/地点/人物/具体场景）
- [ ] 句式长度变化 ≥ 3 种（短/中/长交替）
- [ ] 至少 1 处不完美表达（省略号/自我纠正/犹豫）
- [ ] 结尾不是模板化总结句
- [ ] 整体读感像"给朋友发微信"而非"写文章"

#### 图片去 AI 化

- **信息图模式**（baoyu-xhs-images 生成的排版图）：本质是文字排版图，非 AI 绘画，被检测概率低，可直接使用
- **文字笔记配图**：优先使用场景素材图而非 AI 绘画。如使用 AI 生图，prompt 中必须指定"photo realistic, shot on iPhone, natural lighting"风格，避免典型 AI 画风

### Step 4: 配图生成

<!-- 🔧 修改指引：

#### 【信息图模式】
调用 `baoyu-xhs-images` Skill：
- 将 Step 3 的完整内容草稿作为输入
- **推荐风格**：`study-notes`（学习笔记风）或 `notion`（信息密集风）
- **推荐布局**：`list`（列表型）或 `flow`（流程型）
- 图片比例：3:4（小红书最佳比例）
- 输出 6-9 张信息图

#### 【文字笔记配图】
调用 `gemini-image` 或 `baoyu-image-gen` Skill：
- 根据 Step 3 的配图 prompt 生成
- 图片比例：3:4
- 风格：清新文艺、温暖自然
- 输出 1-3 张配图

### Step 5: 发布前审核

<!-- 🔧 修改指引：
     - 检查清单可根据运营反馈增减项目
     - 预览格式可自定义

展示以下内容供用户确认：

```
📋 发布预览
━━━━━━━━━━━━━━━━━━━━
标题：[标题，≤20字]
形式：[信息图 / 文字笔记]

━━━━━━━━━━━━━━━━━━━━
```

**自动检查清单**：
- [ ] 标题 ≤ 20 字
- [ ] 话题标签 5-8 个
- [ ] 图片 ≤ 9 张
- [ ] 无敏感词（见禁忌词清单）
- [ ] 符合 persona 语气
- [ ] 不重复近 7 天已发内容
- [ ] **反 AI 检测**：通过 Step 3.5 全部 6 项自检
- [ ] **无承接词开头**：全文无"首先/其次/然后/此外/总之"
- [ ] **有私人细节**：至少 2 处个人化场景

**用户操作**：
- 确认发布 → 进入 Step 6
- 要求修改 → 回到 Step 3 调整
- 取消 → 仅保存到本地不发布

| `--pillar` | 指定内容方向 | 自动轮换 |
| `--mode` | `infographic` / `text` | 按方向自动选择 |
| `--style` | 信息图风格 | `study-notes` |
| `--topic` | 手动指定主题 | 自动选题 |
| `--viral` | 爆款复刻模式 | 关闭 |
| `--viral-url` | 指定要复刻的笔记URL | 无 |
| `--dry-run` | 仅生成不发布 | 关闭 |
| `--no-confirm` | 跳过确认直接发布 | 关闭 |

## 使用示例

```bash
# 自动选题，生成并发布
"公考种草"

# 指定方向
"公考种草 --pillar 备考干货"

# 爆款复刻
"公考种草 --viral --viral-url https://www.xiaohongshu.com/explore/xxx"

# 仅生成预览
"公考种草 --dry-run"

# 手动指定主题
"公考种草 --topic '行测资料分析速算技巧'"
```
