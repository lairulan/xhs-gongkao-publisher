## 前置条件

1. xiaohongshu-mcp 服务运行中
2. 已通过 `check_login_status` 确认登录状态
3. 内容已生成完毕（标题、正文、图片、话题）

## 发布流程

### 1. 确认登录

```
MCP Tool: check_login_status
Expected: { logged_in: true, username: "xxx" }
```

如果未登录：
```
MCP Tool: get_login_qrcode
→ 展示二维码给用户
→ 用户扫码后重新检查登录状态
```

### 2. 准备发布内容

**标题处理**：
- 确保 ≤ 20 字
- 去掉首尾空格
- 不包含特殊格式字符

**正文处理**：
- 话题标签附加到正文末尾
- 格式：`正文内容\n\n#话题1 #话题2 #话题3`
- emoji 保持原样（小红书支持 emoji）

**图片处理**：
- 确保图片为本地文件路径（绝对路径）
- 支持格式：JPG、PNG
- 推荐比例：3:4（竖版，小红书最佳）
- 数量限制：1-9 张
- 图片数组按展示顺序排列，第一张为封面

### 3. Browser 发布流程（本地环境，推荐）

Browser 通道优先：零配置、Claude Code 内置 browser 工具直接可用、无需预启动服务。
可通过 `/setup-browser-cookies` 导入真实浏览器 Cookie 实现免扫码。

通过 browser 工具直接操作小红书创作服务平台：

**4.1 打开发布页**
```
browser navigate "https://creator.xiaohongshu.com/publish/publish"
→ 等待页面完全加载（5秒）
→ 确认在"上传图文"标签页
```

**4.2 上传图片（CDP 协议）**
```
通过 CDP DOM.setFileInputFiles 设置图片
→ selector: input.upload-input[type=file]
→ 等待图片上传完成（3-5秒）
→ 截图确认图片缩略图显示
```

**4.3 填写标题（模拟键盘输入）**
```
→ 点击标题输入区域
→ 使用 browser act 逐步输入标题
→ 等待 1-2 秒
→ 确认标题字数计数器更新
```

**4.4 填写正文（模拟键盘输入）**
```
→ 点击正文编辑器
→ 分段输入正文（每段间隔 0.5-1 秒）
→ 确认字数计数器正确
```

**4.5 添加话题**
```
→ 点击 "# 话题" 按钮
→ 搜索话题关键词
→ 从搜索结果中选择官方话题
→ 重复添加 2-3 个话题
```

**4.6 设置为私密发布（关键风控步骤）**
```
→ 滚动到页面底部"内容设置"区域
→ 找到"公开可见"下拉框
→ 尝试切换为"仅自己可见"
→ ⚠️ 已知问题：XHS 的 Vue d-select 组件有反自动化保护，
   合成事件无法触发下拉。如操作失败，以公开模式发布，
   提醒用户在 APP 上手动改为私密
→ 点击"发布"按钮
```

**4.7 确认发布成功**
```
→ 等待页面跳转（URL 含 published=true）
→ 访问笔记管理页确认笔记存在
→ 截图保存发布证据
```

**4.8 提示用户手动检查**
```
发布完成后提示用户：
"笔记已发布，请在手机小红书 APP 上：
 1. 打开笔记预览，检查排版和图片
 2. 如为公开发布，通过 ··· → 权限设置 调整可见性
 3. 同时顺手浏览几篇推荐笔记、点赞 2-3 篇（反检测）"
```

### 4. MCP 发布流程（备用，需预启动服务）

Browser 不可用时，通过 xiaohongshu-mcp 服务发布。需先启动 MCP 服务。

```
MCP Tool: publish_image
Arguments:
  title: "���题文字"
  content: "正文内容（含话题标签）"
  images: ["/path/to/image1.png", "/path/to/image2.png", ...]
  tags: ["公考", "行测", "备考干货"]
```

**MCP 优势**：
- Cookie 自动持久化到 `~/xiaohongshu-mcp/cookies.json`，首次扫码后无需重复登录
- 内置浏览器环境，不受沙箱网络/资源限制
- 直接 API 调用，操作稳定可靠
- 可设置私密发布（无反自动化限制）

### 5. 错误处理

- 检查返回结果
- 如果返回错误：
  - `login_required` → 重新登录
  - `content_blocked` → 检查敏感词，修改后重试
  - `image_error` → 检查图片格式和大小
  - `rate_limit` → 等待 5 分钟后重试

### 6. 记录发布日志

保存到 `output/publish-log.md`，格式：

```markdown
## YYYY-MM-DD HH:MM

- **标题**: xxx
- **方向**: 备考干货
- **形式**: 信息图 / 文字笔记
- **图片数**: N张
- **话题**: #xxx #xxx
- **状态**: 已发布 / 失败（原因）
```

## 风控注意事项

### 发布频率
- 每天不超过 2 篇
- 两篇之间间隔 ≥ 2 小时
- 避免凌晨 0-6 点发布

### 推荐发布时间
| 时段 | 优先级 | 说明 |
|------|--------|------|
| 7:00-8:30 | ⭐⭐⭐ | 早起通勤，备考人群活跃 |
| 12:00-13:00 | ⭐⭐⭐⭐ | 午休时间，浏览高峰 |
| 18:00-19:00 | ⭐⭐⭐ | 下班通勤 |
| 21:00-22:30 | ⭐⭐⭐⭐⭐ | 晚间黄金时段，最佳 |

### 内容安全
- 发布前运行敏感词检查（见 SKILL.md 禁忌词清单）
- 不发布涉及具体考题的内容
- 资料推荐不附带盗版下载链接
- 图片不含其他平台水印

### 异常处理
- 连续 2 次发布失败：停止自动化，提示用户手动检查
- 检测到"操作过于频繁"：暂停 30 分钟
- Cookie 过期：引导用户重新扫码登录

## 行为层反检测（防 AI 托管识别）

小红书 2025 年 3 月起严打 AI 托管账号，除了内容检测，还有**行为模式检测**。

### 操作节奏控制（Browser 发布必须遵守）
- 打开发布页后，等待 3-5 秒再开始操作
- 上传图片后，等待 3-5 秒再填标题
- 填完标题后，等待 2-3 秒再填正文
- 正文分段输入，每段之间间隔 1-2 秒
- 添加完话题后，等待 2-3 秒再设置权限
- 点击发布后，在结果页停留 5-10 秒
- **总操作时长应 >= 2 分钟**（真人至少需要这么久）

### 私密发布策略（推荐）
- 默认以"仅自己可见"模式发布
- 发布后由用户在 APP 上手动检查并切换为公开
- 好处：① 模拟真人草稿-审核-发布流程 ② 给用户最终审核机会 ③ 状态变更记录更自然
- 切换为公开的建议时间：发布后 5-30 分钟

### 发布时间随机化
- **不要**固定每天同一时间发布
- 在推荐时段内随机偏移 10-30 分钟
- 示例：如果目标时段是 21:00-22:30，实际发布时间可以是 21:07、21:43、22:15 等
- 两篇间隔要有变化：有时隔 2 小时，有时隔 4 小时

### 发布后模拟真人行为
发布成功后，建议用户在 10 分钟内执行以下操作（非自动化，手动完成）：
1. 浏览首页推荐 2-3 分钟
2. 给 2-3 篇同领域笔记点赞
3. 在 1 篇笔记下留言互动
4. 回复自己笔记下的评论（如有）

### 内容去同质化
- 连续两天的笔记，方向、形式、标题句式必须不同
- 不要每篇都用相同的 CTA 模板
- 话题标签组合每次至少变化 2-3 个
- 信息图风格不要固定（轮换 study-notes / notion / 其他）

### 账号活跃度维护
- 不要只发不互动（纯发布机器人特征）
- 每天浏览时间 > 发布时间（正常用户行为）
- 偶尔收藏/转发其他人的笔记
